# minamo v0.2.0+ Detailed Design (Road to v1)

**Status**: Design phase（v1 スコープ拡大に基づく。2026-05-30 の v1 設計レビュー）
**Prerequisites**: `docs/concept.md` §5（§5.10〜5.13 追加分）、§11 DEC-018〜024、`docs/roadmap-v1.md`
**Scope**: upcasting / Snapshot / observability hooks / command runner / EventStore facade / RetryExhaustedError / API Extractor gate

## 1. Goals

- concept.md §5.10〜5.13 の型シグネチャに完全準拠する公開 API を追加する（既存 §5.1〜5.9 は不変＝additive）
- 拡大した機能（Snapshot / upcasting）も InMemory / Dynamo の Contract Tests で振る舞い一致を保証する
- 新規 runtime 依存を追加しない（AWS SDK v3 peer + `@microsoft/api-extractor` は devDep のみ）
- thin 原則の維持: hook / interface に留め実装エンジン化を避ける（DEC-019/020/021）

## 2. Release Sequence（DEC-024 = additive / DEC-025 = 単一 v0.2.0 一括）

v1 機能は単一 v0.2.0 で一括リリースする（DEC-025）。下表は実装ユニットと breaking 性の整理であり、当初想定した機能別段階リリース（v0.3=upcasting / v0.4=Snapshot）は採らない。以後の v0.2 → v0.3 → v0.4 は「既存 surface 非破壊」を実証する安定性窓とする（concept.md §12 / roadmap-v1.md §5）。

| 版 | unit | breaking? |
|---|---|---|
| v0.2.0 | U10 observability / U11 runner / U12 facade / U13 RetryExhaustedError / U14 API Extractor gate / NoInfer / U15 upcasting / U16 SnapshotStore / U17 EventStore.loadFrom / U18 executeCommand snapshot 統合 | RetryExhaustedError のみ throw 型変更（1 リリース deprecation）。他は additive（upcast 未指定は identity / snapshot は optional 引数・optional method） |

## 3. Implementation Order（依存グラフ順）

| 順 | Unit | 依存 | 概要 |
|---|---|---|---|
| U10 | §5.12 Observability | U6 | `ExecuteObserver`、executeCommand に observer 配線 |
| U11 | §5.13 createCommandRunner | U6, U10 | executeCommand を config/store 固定で curry |
| U12 | §5.13 createEventStoreTable | U8 | DocumentClient 共有 facade、`.for<TMap>()` narrowing |
| U13 | §5.5 RetryExhaustedError | U6, U10 | retry 枯渇 throw を ConcurrencyError → RetryExhaustedError |
| U14 | API Extractor gate | - | `etc/minamo.api.md` レポート + CI step |
| U15 | §5.11 Upcasting | U6 | `AggregateConfig.upcast`、rehydrate 配線 |
| U16 | §5.10 SnapshotStore | U4 | interface + InMemory/Dynamo 実装、Contract Test |
| U17 | §5.4 loadFrom? | U8, U16 | optional method、Dynamo は version > :v query |
| U18 | §5.6 executeCommand snapshot 統合 | U16, U17 | snapshot load → loadFrom → replay → save |

## 5. Test Strategy

### 5.1 Contract Tests（拡張）
- `test/contract/event-store.ts` — 既存 CT-01〜13 は不変。`loadFrom` を持つ store には CT-14（version > N の部分ロード）を追加
- `test/contract/snapshot-store.ts` — 新規。`save`/`load`/`null on empty`/`overwrite` を InMemory/Dynamo 両方で実行
- InMemory: `test/unit.test.ts` / Dynamo: `test/*.integration.test.ts`

### 5.2 Unit / Type tests
- `test/observability.test.ts` — 各 hook の発火条件（onAttempt / onLoaded / onConcurrencyConflict / onCommitted / onRetryExhausted）を deterministic test double（FailOnce）で検証
- `test/runner.test.ts` — defaults マージ、呼び出し時引数の優先
- `test/event-store-table.test.ts` — `expectTypeOf` で `.for<A>()` と `.for<B>()` が別 TMap に narrow されること（DEC-023 / 型 narrowing gate）
- `test/upcast.test.ts` — 旧 schema イベント列の rehydrate
- `test/snapshot.test.ts` — snapshot からの部分 rehydrate、policy 発火、loadFrom フォールバック

### 5.3 instanceof
- `scripts/verify-instanceof.mjs` に `RetryExhaustedError` を追加
