# minamo v0.1.0 Detailed Design

**Status**: Design phase (Spike 完了後、tag `v0.1.0-spike`)
**Prerequisites**: `docs/concept.md` §5 API Design
**Scope**: concept.md §5.1〜§5.8 の実装、Projection Bridge (§5.7) を含む

## 1. Goals

- concept.md §5 の型シグネチャに完全準拠する公開 API を提供する
- InMemoryEventStore と DynamoEventStore が同一 Contract Test を共有する
- Spike で validate した toolchain / CI / Standard Schema / Release 基盤を基盤とする
- 新規 runtime 依存を追加しない (AWS SDK v3 peer のみ)

## 2. Non-Goals (v0.1.0 スコープ外)

- Cross-aggregate 整合性 (concept.md §6 に記載)
- ProjectionBridge の複数 writer モデル (§5.7 は single-writer 前提)
- API Extractor による breaking change gate (v0.2.0 で導入)
- JSR dual publish (別 PR)
- pre-commit hook 設定 (別 PR)

## 5. Test Strategy

### 5.1 Contract Tests
- `test/contract/event-store.ts` に **同一 suite** を定義
- `test/unit.test.ts` で InMemoryEventStore に対して実行
- `test/dynamodb.integration.test.ts` で DynamoEventStore に対して実行
- 両者に共通する append / load / fresh-read / ConcurrencyError / EventLimitError の契約を網羅

### 5.2 Type-level Regression (expectTypeOf)
- `EventsOf<TMap>` の discriminated union 保持
- `Aggregate<S, E>.state` の `ReadonlyDeep` 適用
- `CommandHandler` の型推論 (TState / TMap / TInput の貫通)

### 5.3 Deterministic Test Doubles
- `FailOnce` / `AlwaysFail` / `CountingLoad` 等を `test/doubles/` に配置
- 時間ベースの `setTimeout` を使わず、呼び出し回数で制御

### 5.4 Standard Schema Integration
- Command input validation を `validate(schema, raw)` で行うサンプル test を追加
- conformant test double で Zod / Valibot / ArkType を模倣

## 6. DEC Candidates (Design Phase で §11 追加可否を判断)

v0.1.0 spike から派生した design decision。concept.md §11 に DEC-015 以降として記録するかは別途レビュー。

- **DEC-015 候補**: Standard Schema v1 による validator 抽象化
  - trigger: Command input validation の vendor lock-in 回避
  - decision: interface のみ受け入れ、実装は consumer 選択
  - rejected alternatives: Zod hard dependency, 独自 validator

- **DEC-016 候補**: Changesets + npm provenance を release 契約に採用
  - trigger: OSS 公開の semver 運用と supply chain 信頼性
  - decision: Changesets で PR ベース semver、npm provenance で SLSA attestation
  - rejected alternatives: semantic-release (設定が CLI driven でない), 手動 publish

- **DEC-017 候補**: typedoc を public API reference の出力ツールに採用
  - trigger: concept.md と実装コードの double source of truth 問題
  - decision: typedoc で TSDoc コメントから自動生成、GitHub Pages に配信
  - rejected alternatives: API Extractor (v0.1.0 は API 未確定で defer), 手動 markdown

## 7. Known Risks

| リスク | 対応 |
|---|---|
| DynamoEventStore の fresh read 要件は consistent read が前提 (§5.4 postcondition) | U8 で `ConsistentRead: true` を明示、integration test で実測検証 |
| EventMap の discriminated union が consumer 側で `satisfies` 漏れる | 型制約で `AggregateConfig.eventMap` を受け、推論を貫通させる |
| ProjectionBridge の strict routing は未実装イベントで compile error にすべき | U9 で `never` fallback を型レベルで強制 |

## 8. Open Questions
